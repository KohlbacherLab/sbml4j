post /network
Request the creation of a new network (createMappingFromMappingWithOptions)

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body NetworkFilterOptions (required)
Body Parameter — <p>The filterOptions to be used in the step upon creation.</p> <p>NodeSymbols are used as follows</p> <ul> <li>in step COPY: ignored, the network gets duplicated with a new entityUUID</li> <li>in step FILTER: Only nodes listed here can be part of the network (and if they match given nodeTypes)</li> <li>in step ANNOTATE: ignored, the fields annotation, annotationName, annotationType are used</li> <li>in step CONTEXT: The first listed geneSymbol is used as the center for the context search</li> <li>in step MULTIGENESUBNET: All listed geneSymbols are used as center for context search, as well as startNode/endNode for dijkstra shortestPath search</li> </ul> <p>NodeTypes are used as follows</p> <ul> <li>in step COPY: ignored, the network gets duplicated with a new entityUUID</li> <li>in step FILTER: Only nodes having one of the listed nodesTypes here can be part of the network (and if they are contained in NodeSymbols)</li> <li>in step ANNOTATE: ignored, the fields annotation, annotationName, annotationType are used</li> <li>in step CONTEXT: ignored (filter the network first, then search context on the newly created network to exlude certain node types)</li> <li>in step MULTIGENESUBNET: ignored (filter the network first, then search context on the newly created network to exlude certain node types)</li> </ul> <p>RelationTypes are used as follows</p> <ul> <li>in step COPY: ignored, the network gets duplicated with a new entityUUID</li> <li>in step FILTER: Only edges having one of the listed relationTypes here can be part of the network</li> <li>in step ANNOTATE: ignored, the fields annotation, annotationName, annotationType are used</li> <li>in step CONTEXT: ignored (filter the network first, then search context on the newly created network to exlude certain relation types)</li> <li>in step MULTIGENESUBNET: ignored (filter the network first, then search context on the newly created network to exlude certain edge types)</li> </ul> <p>NetworkType is set as the networkType property. Allowed Values:</p> <ul> <li>PPI</li> <li>SIGNALLING</li> <li>REGULATORY</li> <li>METABOLIC</li> <li>PATHWAYMAPPING</li> </ul> <p>Annotation is used as follows</p> <ul> <li>in step COPY: ignored</li> <li>in step FILTER: ignored</li> <li>in step ANNOTATE: geneSymbol must match one of the genes in the network, annotation is added to the node under the name defined in annotationName using the type in annotationType</li> <li>in step CONTEXT: ignored</li> <li>in step MULTIGENESUBNET: ignored</li> </ul> <p>AnnotationName is used as follows</p> <ul> <li>in step COPY: ignored</li> <li>in step FILTER: ignored</li> <li>in step ANNOTATE: used as name under which the annotations in the annotation filed of the request body are attached to nodes</li> <li>in step CONTEXT: ignored</li> <li>in step MULTIGENESUBNET: ignored</li> </ul> <p>AnnotationType is used as follows</p> <ul> <li>in step COPY: ignored</li> <li>in step FILTER: ignored</li> <li>in step ANNOTATE: used as type with which the annotations in the annotation filed of the request body are attached to nodes. Is also used to read out the annotation in said format when extracting network</li> <li>in step CONTEXT: ignored</li> <li>in step MULTIGENESUBNET: ignored</li> </ul> <p>minSize is used as follows</p> <ul> <li>in step COPY: ignored</li> <li>in step FILTER: ignored</li> <li>in step ANNOTATE: ignored</li> <li>in step CONTEXT: determines the minimum number of relationships to be traveresed for a path to be included in the context</li> <li>in step MULTIGENESUBNET: determines the minimum number of relationships to be traveresed for a path to be included in the context</li> </ul> <p>maxSize is used as follows</p> <ul> <li>in step COPY: ignored</li> <li>in step FILTER: ignored</li> <li>in step ANNOTATE: ignored</li> <li>in step CONTEXT: determines the maximum number of relationships to be traveresed for a path to be included in the context</li> <li>in step MULTIGENESUBNET: determines the maximum number of relationships to be traveresed for a path to be included in the context</li> </ul> <p>terminateAtDrug is used as follows</p> <ul> <li>in step COPY: ignored</li> <li>in step FILTER: ignored</li> <li>in step ANNOTATE: ignored</li> <li>in step CONTEXT: determines whether the context search should only contain paths (within minSize, maxSize) that end in a 'Drug'-Node. It is required to add MyDrug annotations to a network for this to work.</li> <li>in step MULTIGENESUBNET: determines whether the context search should only contain paths (within minSize, maxSize) that end in a 'Drug'-Node. It is required to add MyDrug annotations to a network for this to work.</li> </ul>

Request headers

Query parameters

networkEntityUUID (required)
Query Parameter — The entityUUID of the network to get format: uuid
step (required)
Query Parameter — <p>The type of step to be performed upon creation (one of: COPY, ANNOTATE, FILTER, CONTEXT, MULTIGENESUBNET)</p> <p>Step determines which action to perform upon network creation.</p> <ul> <li>COPY: duplicate the given network with all it's contained nodes and edges</li> <li>FILTER: duplicate the given network but filter according to properties in the Request body (see sections of request body fields for details)</li> <li>ANNOTATE: duplicate the given network and add annotation to nodes using the fields 'annotation', 'annotationName', 'annotationType'</li> <li>CONTEXT: calculate and persist the network context in given network for given gene with request body parameters 'minSize', 'maxSize', 'terminateAtDrug'</li> <li>MULTIGENESUBNET: calculate and persist a subnetwork from the given network containing the contexts around the given genes according to the paramters found in CONTEXT as well as the pairwise shortest paths (using dijkstra with constant edge-score) between the genes in the given network</li> </ul>
